# Weld 데모

진짜 임시 git 저장소에서 실제 `git merge`를 돌려 Weld 머지 드라이버가 무엇을
하는지 보여주는 end-to-end 데모. 부품별 테스트와 달리 파이프라인 전체
(분류 → 후보생성 → 검증 → 판정 → 에스컬레이션)를 실제 git 위에서 이어 돌린다.

## 실행

```bash
pip install -e .          # weld 명령이 PATH에 있어야 한다 (git이 이걸 호출)
python demo/run_demo.py
```

## 무엇을 보여주나

- **시나리오 ① 가짜 충돌**: A와 B가 서로 다른 함수를 건드림 → mergiraf가 구조적
  으로 자동 병합 → Weld가 사람 개입 없이 자동 커밋. **현재 완전 동작.**
- **시나리오 ② 진짜 충돌**: A와 B가 같은 줄을 다르게 수정 → 검증 게이트로.
  `candidates`/`policy` 파트가 아직 stub인 동안에는 표준 충돌 마커로 안전하게
  폴백한다.

## "살아있는 데모"

같은 스크립트가 파트가 들어올 때마다 더 많이 작동한다. 이서영님의
`candidates`/`policy`/`escalate`가 구현되면 시나리오 ②도 실제 LLM 후보 생성 →
검증 → 자동병합/에스컬레이션까지 돈다. **팀원 파트를 pull 받은 직후 이 데모를
다시 돌려 통합이 됐는지 즉시 확인**하는 용도이자, 발표 시연용.
